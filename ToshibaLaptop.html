Linux on the Toshiba Satellite A45S150 Laptop


I recently bought a Toshiba Satellite A45S150 and installed Linux on it (spent a few weeks off and on and a little frustration searching and finding, drinking coffee, etc.)

I didn't see anything for Linux out there on the A45S150 and thought I'd contribute since I got almost everything (the basics) working.  I suggest sitting down (hopefully wireless is working on your WinXP setup and you can read this on a comfortable couch) and reading this all the way through to save any frustration/problems I may have already solved.

Take some time to figure out your partition scheme and what you want to do with Linux and/or which distribution you want to use.  Note I got just the "basics" working (i.e., enough to setup Linux for web-surfing and C/C++ software development from the couch via wireless).

Feel free to email me (dperique@yahoo.com) with comments and/or suggestions (which I may or may not implement -- I have a day job that really takes up most of my days (and nights sometimes)).

The Toshiba Satellite A45S150 comes with the following configuration (and then some):
  • Pentium 4 2.4Ghz
  • 60G disk (one partition) for WinXP Home
  • 512M RAM
  • 10/100 Ethernet (wired), 802.11g (54M) wireless (802.11b compatible)
  • Intel 82852/82855 GM/GME Graphics controller (32M RAM) 1024x768
  • SoundMax Integrated Digital Audio (i8xxx)
  • USB 2.0 x 4; TV out (s-video)
  • etc. (see advertisements from complete list)
Here's what I setup and found out:
  • My partition scheme
    • Bought Partition Magic 8.0 from BestBuy via download for ~$50
    • Chop up the 60G partition as follows
      • 25G WinXP (primary NTFS)
      • 33.5G Extended
        • 500M FAT32 for sharing between Linux and WinXP
        • 8.2G ext3 for the root partition (mounted as /)
        • 2.0G ext3 for user files (mounted as /home)
        • 522M swap (this laptop comes stock with 512M RAM)
        • 22.2G NTFS for my WinXP data
    • Note that I have /home on a separate partition; this was so I can install/reinstall Linux (during my experimental stage) without having to blow away my user data.
    • I left enough space on WinXP for applications and data and created 500M of FAT32 so I can share files between WinXP and Linux since NTFS support on Linux is read-only.
  • Dual boot information
    • I use lilo on the first partition /dev/hda1
    • I tried grub on Redhat 8.0 and it worked but I don't know much about it
  • Redhat 8.0 (I installed it because I had the CDs laying around)
    • Sound card works after using redhat-configsoundcard
    • XFree86 does not work because you need XFree86 4.3 (RH8.0 uses 4.2)
    • 10/100 Ethernet (onboard) is recognized and works just fine
    • Atheros 802.11g wireless (onboard) is not recognized
    • You can rebuild the kernel and get NTFS support and it works (readonly)
  • Mandrake 9.1 Standard (was hoping this would work)
    • Bought from MandrakeSoft for about $33
    • Sound card worked
    • XFree86 worked (Mandrake 9.1 comes with XFree86 4.3)
    • 10/100 Ethernet (onboard) worked
    • Atheros 802.11g wireless (onboard) is not recognized
    • Could not find gcc or kernel so I could do nothing here
  • Redhat 9.0 (finally settled on this distribution)
    • Bought the CDs from getlinuxcds.com for about $9
    • Sound card works after using redhat-configsoundcard
    • XFree86 works (RH9.0 comes with XFree86 4.3; note that I had to run redhat-config-xfree86 and restart X (before I did that I got some fuzzy lines on the screen and felt I was somehow frying the screen); upon a later reboot, I could see on the boot messages that Linux was recognizing my graphics card.  I use the touchpad or a USB wheel mouse; the switch, you must use redhat-config-mouse and restart X.
    • 10/100 Ethernet (onboard) is recognized and works just fine
    • Atheros 802.11g wireless (onboard) is not recognized; you have to go to the "madwifi" site (do a google search) and get the drivers.  I did an anonymous CVS checkout and built it; read the faq.  Once built and loaded the 802.11g was found and came up with no problems (note that you need sharutils to get the ath_hal.o to work; I used sharutils-4.2.tar.gz)
    • You can rebuild the kernel and get NTFS support and it works (readonly).
  • Other:
    • I have an old (1 year) 802.11b Linksys card for PCMCIA.  I had to use pcmcia 3.2.7 to get it to work:
      • Download pcmcia 3.2.7
      • cd /usr/local ; tar xzf pcmcia-3.2.7.tar.gz
      • do a 'make' (I took all defaults when asked questions)
      • do a 'make install'
      • do /etc/rc.d/init.d/pcmcia restart (to restart PCMCIA services)
      • do 'tail -f /var/log/messages'
      • Create and/or modify /etc/sysconfig/network-scripts/ifcfg-eth0 or something like that; I set it to DHCP
      • insert the card (hopefully hear two high pitched beeps and see messages that look good)
    • Useful commands on Redhat
      • redhat-config-packages
      • redhat-config-sound
      • redhat-config-time
      • redhat-config services (e.g. so you can turn on FTP)
      • redhat-config-securitylevel
    • Turn on some useful services
      • In /etc/xinet.d/telnet (set disable to no)
      • In /etc/xinet.d/vxftp (set disable to no)
      • /etc/rc.d/init.d/xinetd restart (to restart inet daemon)
    • Note that if you want to build a kernel, start with the config in /boot called config-2.14-18 (in RH 8.0) so that you can only slightly modify your stock RH kernel (i.e., minimize chances of messing up something that already works).
    • Here's what I do for the Atheros 802.11g setup:
      • cd /usr/local
      • tar xzf madwifi.tar
      • install sharutils or do 'alias uudecode="perl uudecode.pl"')
      • cd ./madwifi
      • make (make sure there are no errors on the build)
      • insmod wlan/wlan.o
      • insmod ath_hal/ath_hal.o
      • insmod driver/ath_pci.o
      • make install (puts the .o files in /lib/modules...)
      • ifup ath0 up (after you created /etc/sysconfig/network-scripts/ifcfg-ath0)
      • iwpriv ath0 mode 3 (lock opertion to 11g (54M)
      • See the madwife README for more tools/information
  • I did not get the APM stuff to work.  I get "apm: BIOS not found" during bootup.   I'd appreciate any help with this.  I just want to extend the life of a battery charge (automatically shutoff disks, show me how much charge I have left, etc.) and be able to turn off the laptop without having to press the power button).
